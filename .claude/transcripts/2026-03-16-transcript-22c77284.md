## What "Thinking" means technically

When you see "Thinking...", I'm using **extended thinking** — a separate, internal reasoning step that happens *before* I produce my visible response.

### The two-phase process

**Phase 1 — Thinking (hidden by default)**
- I generate tokens into a **thinking block** — a scratchpad that is *not* shown to you in the final output (though Claude Code can be configured to show it).
- This is literally me generating text, same mechanism as my visible reply — there's no separate "reasoning engine." It's autoregressive token generation, just routed to a different output buffer.
- I use it to: break down the problem, consider approaches, plan tool calls, weigh tradeoffs, self-correct before committing to an answer.

**Phase 2 — Response (visible)**
- After the thinking block closes, I generate the actual response you see — text, tool calls, etc.
- The thinking content stays in my context window, so my visible response is informed by it, but you only see the final output.

### Concrete dry run

Say you ask: *"Refactor this function to use async/await"*

```
1. Your message arrives
2. I enter thinking mode:
   ┌─ thinking block (hidden) ──────────────────────┐
   │ "User wants async refactor. Let me read the     │
   │ function first. It uses .then() chains with      │
   │ error handling. I need to preserve the catch      │
   │ semantics — use try/catch. The return type        │
   │ changes to Promise<T>. Should I also update       │
   │ callers? User didn't ask for that, keep scope     │
   │ tight. I'll use the Read tool first."             │
   └───────────────────────────────────────────────────┘
3. Thinking block ends
4. Visible response begins: tool calls (Read), then the edit
```

### When does thinking trigger vs. not?

It's controlled by a **reasoning effort** parameter (low/medium/high) and the model's own judgment about complexity. Simple factual questions or short replies may skip it entirely. Complex multi-step tasks, ambiguous requests, or anything requiring planning tends to trigger it.

In this conversation, my reasoning effort is set to roughly medium-high — so I'll think on most substantive questions but may skip it for trivial ones.

### What it is *not*

- It's **not** a different model or a separate system — same weights, same architecture.
- It's **not** a search process or tree-of-thought — it's linear token generation, just like my visible text.
- It's **not** guaranteed to be correct — I can reason poorly in the thinking block too. But having the scratchpad space to "think out loud" before committing significantly reduces errors.

The key insight: "Thinking" is just me writing to myself before writing to you. Same mechanism, different destination buffer.
