# App Icon Polish — Squircle Container + Depth

## Context
The icon currently fills the entire canvas with an opaque black background (`fillRect` before the squircle). In the macOS Dock, dev-mode apps are **not** automatically masked to a squircle by the OS — only properly signed/bundled `.app`s get that treatment. The result: the raw PNG is shown as a plain dark square.

The correct approach for macOS icons is to use **alpha-channel transparency** in the corner areas so the icon's own pixel data defines the rounded shape — no dependency on OS masking. The squircle renders on a transparent canvas, corners are transparent, and the icon looks correct everywhere (Dock dev mode, Finder, mission control, etc.).

One change needed:
- **Remove the black `fillRect`** — let the canvas stay transparent outside the squircle
- **Tighten the drop shadow** (blur 40, offsetY 8) so it fits within the 30px margin without being clipped

---

## Single change to `generate.mjs`

Remove the black canvas fill:
```js
// DELETE these two lines:
ctx.fillStyle = '#000000';
ctx.fillRect(0, 0, SIZE, SIZE);
```

Tighten drop shadow to fit within 30px margin:
```js
ctx.shadowBlur    = 40;   // was 55
ctx.shadowOffsetY = 8;    // was 12
```

Everything else (squircle path, clip, background gradient, brain, nodes, rim stroke) is unchanged.

The PNG output will have **transparent alpha** in the corner areas — the rounded shape is defined by the pixel data itself, not by OS masking. This makes it look correct in dev mode Dock, Finder, mission control, everywhere.
