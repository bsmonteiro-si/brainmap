## Assistant

**Token economics:**
- Thinking tokens count against the output token budget, but they're billed separately from visible output tokens
- There's a configurable thinking budget — Claude Code sets this as part of the API call
- If I run out of thinking budget, I stop reasoning and start producing visible output with whatever conclusions I've reached so far

**Streaming behavior:**
- The API streams thinking tokens as a distinct content block type (`thinking` type vs `text` type)
- Claude Code receives these in real-time and shows the "Thinking" animation/indicator
- Once the thinking block closes and a `text` block starts streaming, the UI transitions from "Thinking" to showing my actual response character by character
- Some implementations show a summary or truncated version of thinking content — Claude Code currently just shows the indicator

**Context window implications:**
- Thinking tokens from previous turns are **discarded** before the next turn — they don't accumulate in the conversation history
- This is a key design choice: it means thinking doesn't eat into the context window over a long conversation
- My visible response *is* kept in context, so any conclusions I need to remember must be in the visible output
- This is why sometimes I'll write notes to myself in the visible response or use tools to persist information — the thinking is gone next turn

**What I can and can't do in thinking:**
- I *can*: reason, plan, reconsider, do math, weigh options, draft and revise, catch my own mistakes
- I *can't*: call tools from inside thinking. Tool calls only happen in the visible output. So if I need to read a file to reason about it, I have to call the tool visibly first, then think about the results in a subsequent turn
- I *can't*: see my own previous turns' thinking. Each turn's thinking is independent

**How it differs from "no thinking":**
- Without thinking, I go straight to generating visible output token by token — my first token commits me to a direction
- With thinking, I can explore multiple approaches, realize one is wrong, and pivot — all before the first visible token
- This is why thinking tends to produce better results for complex tasks: it's not just more compute, it's *revisable* compute

**The "Thinking" display in Claude Code specifically:**
- Claude Code uses a streaming renderer that detects the content block type
- During thinking, it shows an animated indicator (the pulsing "Thinking" text)
- The actual thinking content *is* received by the client but typically not shown to the user in the default view
- Some configurations or modes may let you see the thinking content, but by default it's hidden to keep the UI clean
