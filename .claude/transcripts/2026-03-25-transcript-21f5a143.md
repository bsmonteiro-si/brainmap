## Assistant

Good question. Here's why the duplication exists and what unification would look like:

**Why it happens:**

React Flow internally uses its own Zustand store. When you pass `nodes` and `edges` as props, React Flow *copies* them into its internal state. From that point, there are two copies:

1. **Our state** — whatever we hold in React state/store to represent the `.canvas` data model
2. **React Flow's internal state** — what it actually renders and lets users drag/resize/select

Every user interaction (drag, resize, type) updates React Flow's copy first, then fires callbacks (`onNodesChange`, `onEdgesChange`) where we're supposed to sync back to our copy. Our copy is what gets serialized to disk on auto-save.

The bugs come from the gap between these two: if our sync callback misses something, applies constraints differently, or if a re-render pushes stale data from our state back into React Flow's props, you get the flickering/regression behavior.

**Can we unify them?**

Yes, partially. React Flow supports a **controlled** and an **uncontrolled** pattern:

- **Controlled** (what we likely do now): We pass `nodes={ourNodes}` and handle `onNodesChange` to update our state. This creates the two-copy problem — every frame, React Flow compares our props to its internal state.
- **Uncontrolled**: We use React Flow's `useReactFlow()` hook and `useStoreApi()` to read/write directly to React Flow's internal store. We'd only serialize *from* React Flow's store when saving to disk. No second copy.

The uncontrolled approach would mean: React Flow's store is the single source of truth at runtime, and the `.canvas` file is just the serialized snapshot. We'd translate JSON Canvas ↔ React Flow format on load/save only, not on every render.

**The trade-off:** We lose the ability to externally drive node state (e.g., updating a node's content from outside the canvas). But that might not be something we need much of.

Before I commit to recommending this, I want the agents to verify what pattern we actually use today and how deep the controlled-mode coupling goes. Want me to proceed with the broad exploration across all 5 areas, with special attention to the state ownership question?
