# Fix: Interface Font Size Has No Effect

## Context
The "Interface Font Size" slider in the Settings modal changes `--ui-font-size` and sets `body { font-size: var(--ui-font-size) }`. However, 54 of the 56 `font-size` declarations in App.css use hard-coded pixel values (e.g., `font-size: 12px`) that override CSS inheritance from `body`. As a result, the slider has no visible effect on any UI panel.

## Approach
Introduce three derived CSS variables alongside the existing `--ui-font-size` base, then replace hard-coded `font-size` values in content-bearing UI selectors with the appropriate variable. Pure CSS change — no TypeScript/React changes needed.

---

## New CSS Variables
**File:** `crates/app/src/App.css` — add to `:root` block (after `--editor-font-size`)

```css
--ui-font-lg: calc(var(--ui-font-size) + 1px);   /* 14px at default 13px */
--ui-font-sm: calc(var(--ui-font-size) - 1px);   /* 12px at default 13px */
--ui-font-xs: calc(var(--ui-font-size) - 2px);   /* 11px at default 13px */
```

These cascade automatically: when the user sets `--ui-font-size` to 16px, `--ui-font-sm` becomes 15px, `--ui-font-xs` becomes 14px, etc.
