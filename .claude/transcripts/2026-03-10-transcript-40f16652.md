### 1b. Nodes — smaller, glowing, labels auto-hide

```ts

    height: 8,
```

## Step 2: `GraphView.tsx` — Node sizing + fcose params

### 2a. Node sizing formula — smaller base, smaller increment

```ts
// Current:
const size = Math.max(20, 20 + n.indegree(false) * 4);

// New:
const size = Math.max(6, 6 + n.indegree(false) * 2);
```

This gives: 0 in-edges → 6px, 5 in-edges → 16px, 10 in-edges → 26px — proportionate to the new small-node aesthetic.

### 2b. fcose layout — more spread, lower gravity

```ts
cy.layout({
  name: "fcose",
  // animate only in the layout-switch effect (where the user explicitly changed layout),
  // NOT here — this function is called from the data-sync effect too, and animating
  // on every node/edge update would reset pan/zoom on every note interaction.
  animate: false,
  quality: "proof",
  idealEdgeLength: 180,     // was 120 — more separation
  nodeRepulsion: 18000,     // was 8000 — stronger push
  gravity: 0.15,            // was 0.25 → lower, but not so low isolated nodes drift (0.08 was too low)
  fit: true,
  padding: 60,              // was 40
  nodeDimensionsIncludeLabels: false,  // was true — labels hidden at default zoom
} as cytoscape.LayoutOptions).run();
```

Animation (Obsidian's settling feel) is limited to the initial load by calling `runLayout` once with `animate: true` from the Cytoscape `useEffect([], [])` init effect, AFTER the initial data is applied — **not** from the ongoing data-sync effect. Implementation note: the current code structure has `runLayout` called only from the data-sync `[nodes, edges]` effect. To animate only on first load, pass an `isFirstLoad` flag (using a `useRef<boolean>`) and set `animate: isFirstLoad` in `runLayout`.

### 2c. Update Cytoscape init — set background color on container

Cytoscape renders on a `<canvas>` element; the container background shows in the margins. Cytoscape reads the container's `background-color` for the canvas itself when initialized. We can set it directly:

```ts
const cy = cytoscape({
  container: containerRef.current,
  style: graphStylesheet,
  layout: { name: "preset" },
  minZoom: 0.1,
  maxZoom: 5,
  wheelSensitivity: 0.3,
  // No backgroundColor property needed — handled via CSS on container
});
```

No change to the init call itself; the CSS handles it.

## Step 3: `App.css` — Dark graph container background

```css
.graph-container {
  width: 100%;
  flex: 1;
  min-height: 0;
  background: #13131a;   /* ← always dark, intentional — matches Obsidian graph aesthetic */
}
```

This is intentionally hardcoded rather than a CSS variable. The graph canvas should always be dark regardless of the app's light/dark theme toggle (Obsidian behaves this way too — the graph is always rendered on a dark background). The Cytoscape canvas `<canvas>` element takes the `background-color` of its parent `.graph-container`, so this single rule covers both.
