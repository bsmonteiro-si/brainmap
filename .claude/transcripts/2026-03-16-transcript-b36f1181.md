## Assistant



The core issue: `createHighlight` reads `window.getSelection()` live at click time, but the selection gets cleared between mousedown and click. The `onMouseDown preventDefault` pattern isn't reliable enough across webviews. Additionally, `checkSelection` only listens for `mouseup`/`keyup` inside the scroll container, missing selection changes from other sources.

**Fix**: Snapshot the selection data (text, page number, rects) eagerly when the selection is first detected, then use that snapshot in `createHighlight` instead of re-reading the live selection. Also switch to the `selectionchange` event for more reliable tracking.
