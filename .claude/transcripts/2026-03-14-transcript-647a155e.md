## Assistant

Found the bug. Here's the sequence:

1. Click expand → `toggleFocusMode()` → `leftPanelCollapsed: true`
2. `useEffect` calls `contentPanelRef.current?.collapse()` → panel collapses to 0%
3. `onLayoutChanged` fires → `savePanelSizes` stores `{ content: 0, editor: 100 }` for the active tab
4. Click expand again → `toggleFocusMode()` → `leftPanelCollapsed: false`
5. First `useEffect` calls `expand()` → panel starts expanding
6. **Second `useEffect` also fires** (since `leftPanelCollapsed` is in its deps and is now `false`) → calls `resize("0%")` using the saved collapsed size → **immediately re-collapses the panel**

The `onLayoutChanged` callback saves the collapsed size (0%) into `panelSizes`, and the resize effect then uses that stale 0% to undo the expand.
