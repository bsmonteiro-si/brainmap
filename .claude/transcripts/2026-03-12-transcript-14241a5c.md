# AI-First Structured Logging for BrainMap

## Context

BrainMap is an AI-first app, but has zero logging infrastructure — only scattered `eprintln!` (Rust) and `console.error()` (TS). When the tooltip was broken, we had to add a throwaway debug command and file writer just to see what was happening. AI agents need structured, persistent, easily-parseable logs to diagnose issues, understand app behavior, and operate effectively.

## Current State

| Layer | Mechanism | Structured? | Persistent? |
|-------|-----------|-------------|-------------|
| Core | `eprintln!` warnings only | No | No |
| CLI | Error envelope on stdout | Partially | No |
| MCP | Error envelope over stdio | Partially | No |
| Tauri | `eprintln!` in watcher | No | No |
| Frontend | `console.error()` inline | No | No |

No log levels, no log files, no env var config, no documentation.

## Design

### Schema (NDJSON, one JSON object per line)
```json
{"ts":"2026-03-12T10:30:00.123Z","level":"INFO","target":"brainmap_core::workspace","msg":"workspace opened","fields":{"root":"/path","node_count":34,"edge_count":52}}
```

### Log Destinations
- **File**: `<workspace>/.brainmap/logs/brainmap.log` (daily rotation, 3 days retained)
- **stderr**: JSON for MCP (agents read it), human-readable for CLI
- **stdout**: Never (reserved for MCP protocol / CLI output)
- **Frontend**: `console` methods with `[brainmap] {JSON}` prefix

### Configuration
- Env var `BRAINMAP_LOG` using `tracing` `EnvFilter` syntax
- Defaults: CLI=`warn`, MCP=`info`, Tauri=`info`
- Examples: `BRAINMAP_LOG=debug`, `BRAINMAP_LOG=brainmap_core::workspace=trace`

### What to Log

| Level | What |
|-------|------|
| **error** | Corrupted index, SQLite failures, file I/O errors, unrecoverable state |
| **warn** | Broken links during parse, invalid YAML (note skipped), watcher errors |
| **info** | Workspace open/close, note/link CRUD, search queries, MCP tool calls, config changes |
| **debug** | Parse results, graph diffs, index queries, watcher events, subscriber config |
| **trace** | Edge computation, FTS5 queries, graph traversal (hot paths) |
