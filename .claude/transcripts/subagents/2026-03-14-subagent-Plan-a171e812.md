## Implementation Plan: "Raw" View Mode for BrainMap Editor

### Overview

Add a third view mode ("Raw") to the editor that displays the complete on-disk file content including YAML frontmatter in a read-only CodeMirror instance. This applies only to BrainMap notes (files with frontmatter parsed by the core). Plain files and untitled tabs already show raw content, so the Raw button is hidden for them.

### Key Design Decisions

1. **Read-only**: Raw mode uses a read-only CodeMirror. This is the simplest and safest approach -- editing raw YAML frontmatter could easily corrupt the data model. Users who want to edit frontmatter use the structured FrontmatterForm; users who want to edit body use Edit mode.

2. **Reuse `readPlainFile` for raw content**: The existing `read_plain_file` Tauri command reads any file as raw UTF-8 text with no special parsing. It works on BrainMap `.md` files and returns the full content including YAML frontmatter. No new Rust command is needed.

3. **Fetch on switch, not cached**: Raw content is fetched each time the user switches to Raw mode. This guarantees it always reflects the on-disk state (including after saves). The content is stored in a `rawContent` field in the editor state for display but is discarded on mode switch away. This avoids staleness issues and keeps memory usage low.

4. **No persistence of Raw mode in tabs**: When switching tabs, if a tab was in Raw mode, it can be restored to Raw mode (the `viewMode` field already persists per-tab), but the `rawContent` will be re-fetched on activation.
