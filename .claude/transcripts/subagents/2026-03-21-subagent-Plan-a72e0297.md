### Overview

This feature adds a context menu option on plain `.md` files in the Files panel to convert them into full BrainMap notes by prepending YAML frontmatter. ### Step 1: Core -- `Workspace::convert_to_note()` in workspace.rs

**File:** `/Users/bsmonteiro/Desktop/Personal/brainmap/crates/core/src/workspace.rs`

Add a new public method after `create_note` (around line 385):

```rust
pub fn convert_to_note(
    &mut self,
    rel_path: &str,
    note_type: Option<&str>,
) -> Result<RelativePath> {
```

Logic:
1. Construct `RelativePath` from `rel_path`.
2. Validate the file is NOT already a note: `if self.notes.contains_key(&path) { return Err(BrainMapError::DuplicatePath(...)) }`.
3. Validate the file exists on disk and has `.md` extension.
4. Read the existing file body from disk via `std::fs::read_to_string`.
5. Derive the title from the filename stem (e.g., `Path::new(rel_path).file_stem()` then replace hyphens/underscores with spaces).
6. Build a `Frontmatter` struct:
   - `id: NoteId::new()`
   - `title`: derived from filename
   - `note_type`: `note_type.unwrap_or("concept")` 
   - `tags: vec![]`, `status: None`, `source: None`, `summary: None`, `links: vec![]`, `extra: HashMap::new()`
   - `created` / `modified`: `Local::now().date_naive()`
7. Build a `Note` struct with the existing body content and empty `inline_links` (the parser will re-derive inline links when `parse_file` runs on reload).
8. Serialize via `parser::serialize_note(&note)`.
9. Write back to the same file path (`std::fs::write`).
10. Add to index: `self.index.add_note(&note, file_mtime(&file_path))`.
11. Add node to graph with `self.graph.add_node(...)`.
12. Call `self.ensure_folder_nodes(&path)` for folder hierarchy.
13. Insert into `self.notes`.
14. Call `self.graph.assert_invariants()`.
15. Return `Ok(path)`.

This closely follows the pattern of `create_note` (lines 319-385) but reads existing body from disk instead of taking it as a parameter, and does NOT create the file (it already exists).

### Edge Cases and Considerations

1. **File already has partial/malformed frontmatter:** The `convert_to_note` method checks `!self.notes.contains_key()` -- if the file was already parsed as a note during workspace load, it would already be in the map. If the frontmatter was malformed and the file was NOT parsed, the conversion will prepend fresh frontmatter. The old malformed YAML would end up in the body, which is acceptable (the user can clean it up).

2. **Binary or empty files:** The `.md` extension check in the core should be sufficient. Reading a binary `.md` file would produce garbled body text, but this is a user error.

3. **Concurrent access:** The `with_slot_mut` locking pattern in the Tauri command ensures exclusive access during conversion.

4. **File watcher:** `register_expected_write` prevents the watcher from re-triggering a `reload_file` for the same write.

5. **Title derivation:** Use `file_stem` and replace `-` and `_` with spaces. This matches the convention established by `CreateNoteDialog`.
