### Analysis

The problem has two parts:

1. **Remove the auto-open of CreateNoteDialog** (trivial - delete line 364)
2. **Make empty folders visible in the tree** (requires tracking created folders)

For part 2, the three options have clear trade-offs:

- **Option A (local component state)**: Simplest. A `Set<string>` in `FileTreePanel` tracking folder paths created during this session. Lost on remount, but that is acceptable since empty folders persist on disk and a "list dirs" command could be added later.
- **Option B (Tauri command to list directories)**: Most robust, but requires Rust-side changes and a new API endpoint. Over-engineered for this fix.
- **Option C (store-level tracking)**: Persists across component remounts but adds complexity to graphStore/uiStore for a concept (empty folders) that is orthogonal to both.

**Recommendation: Option C (uiStore)** is the best balance. It survives component remounts (e.g., when switching between Graph/Files tabs, which uses CSS display toggle but could change), and uiStore already tracks tree-related state like `treeExpandedFolders`. The set does not need localStorage persistence -- it only needs to last for the session since empty folders exist on disk.

However, there is a subtlety: when a note is created inside a tracked empty folder, that folder should be removed from the tracked set (it is no longer empty since `buildTree` will now create it from the note's path). And when a folder is deleted, it should also be removed from tracking.
