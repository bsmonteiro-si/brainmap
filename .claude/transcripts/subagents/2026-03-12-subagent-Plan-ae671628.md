## Implementation Plan: "Create & Link" in LinksEditor

### Overview

When a user types a note title in LinksEditor's target input and no existing note matches, show a "Create & Link" button that opens CreateNoteDialog pre-filled with the typed title. After note creation, automatically link the new note to the current note with the selected relationship type, without navigating away.

### Design Decision: Callback via UIStore

The cleanest approach is to extend UIStore with two new optional fields that CreateNoteDialog can check:

- `createNoteInitialTitle: string | null` -- pre-fills the title field (separate from path)
- `createNoteOnCreated: ((createdPath: string) => void) | null` -- callback invoked after successful creation, which also suppresses the default `openNote` behavior

This keeps the wiring centralized in UIStore (the existing pattern for dialog state), avoids prop-drilling, and requires minimal changes. The callback closure captures the LinksEditor's `notePath` and `newRel` values at the time "Create & Link" is clicked.

### Edge Cases & Considerations

- **Callback error handling**: If `createLink` fails inside the callback, the note is still created (which is correct -- the user can manually link later). The error is logged to console. We could show a toast, but that's out of scope for this feature.
- **Race condition**: The user could close/switch the current note between clicking "Create & Link" and the dialog completing. The callback captures `notePath` at click time, so the link is created correctly even if the editor moves on. `refreshActiveNote` will be a no-op if the active note has changed.
- **Input clearing**: `setNewTarget("")` is called immediately when "Create & Link" is clicked, so if the user cancels the dialog, the input is already clear. This is acceptable UX.
- **Zustand serialization**: `createNoteOnCreated` is a function stored in Zustand. This is fine since Zustand doesn't serialize state by default, and this field is transient (cleared on dialog close). It is not persisted to localStorage.
- **Dialog cancel**: If the user cancels CreateNoteDialog, `closeCreateNoteDialog` clears the callback. No link is created. Clean.
