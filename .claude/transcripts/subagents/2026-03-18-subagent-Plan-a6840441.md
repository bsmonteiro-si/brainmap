## Implementation Plan: Mermaid Diagram Rendering in CodeMirror 6

### Overview

Add live mermaid diagram rendering to the CodeMirror editor using the same cursor-aware replace-widget pattern used by the `TableWidget`. When the cursor is outside a `mermaid` fenced code block, the raw markdown is replaced with a rendered SVG. When the cursor enters the block, the raw source is shown for editing. Also add rendering in MarkdownPreview and a `/mermaid` slash command.

### Architecture Decision: Separate File

Create a new `cmMermaidDecorations.ts` file rather than extending `cmMarkdownDecorations.ts`. Rationale:
- The mermaid extension has its own async rendering concern (mermaid.render is async), its own widget class, and its own error-handling logic.
- `cmMarkdownDecorations.ts` is already 527 lines and handles 7 different decoration types.
- Every other major decoration type has its own file (`cmCalloutDecorations.ts`, `cmCheckboxDecorations.ts`, `cmBulletDecorations.ts`, `cmListSpacing.ts`).
- The new file will import and reuse `scanFencedBlocks` from `cmMarkdownDecorations.ts`.

Add `"mermaid": "^11.6.0"` to `dependencies`. The mermaid package is the standard library (maintained by the mermaid-js team, 70k+ GitHub stars). Version 11.x supports ESM imports and has a `mermaid.render()` API that returns SVG strings.

### Step 2: Extend FencedBlock to capture the language

#### 3a: Mermaid initialization

```typescript
import mermaid from "mermaid";

let mermaidInitialized = false;
function ensureMermaidInit() {
  if (mermaidInitialized) return;
  mermaid.initialize({
    startOnLoad: false,
    theme: "default", // or detect from effectiveTheme
    securityLevel: "strict",
  });
  mermaidInitialized = true;
}
```

The `securityLevel: "strict"` prevents XSS from diagram content. `startOnLoad: false` ensures mermaid doesn't auto-scan the DOM.

#### 3b: SVG render cache

Mermaid rendering is expensive (parses the diagram DSL, generates SVG). Use a `Map<string, string>` cache keyed by the mermaid source text. On cache miss, render asynchronously and schedule a CM view update when the SVG is ready.

```typescript
const svgCache = new Map<string, { svg: string } | { error: string }>();
let pendingRenders = new Set<string>();

async function renderMermaid(source: string, view: EditorView): Promise<void> {
  if (svgCache.has(source) || pendingRenders.has(source)) return;
  pendingRenders.add(source);
  ensureMermaidInit();
  try {
    const id = `mermaid-${Date.now()}-${Math.random().toString(36).slice(2)}`;
    const { svg } = await mermaid.render(id, source);
    svgCache.set(source, { svg });
  } catch (e) {
    svgCache.set(source, { error: e instanceof Error ? e.message : String(e) });
  } finally {
    pendingRenders.delete(source);
  }
  // Trigger a CM update so the decoration rebuilds with the now-cached SVG
  view.dispatch({ effects: mermaidRenderEffect.of(null) });
}
```

Use a `StateEffect` (`mermaidRenderEffect`) to signal the StateField to rebuild decorations after an async render completes.

#### 3c: MermaidWidget

```typescript
class MermaidWidget extends WidgetType {
  constructor(
    readonly source: string,
    readonly cached: { svg: string } | { error: string } | undefined,
  ) {
    super();
  }

  eq(other: MermaidWidget): boolean {
    return this.source === other.source;
  }

  toDOM(): HTMLElement {
    const wrapper = document.createElement("div");
    wrapper.className = "cm-mermaid-widget";

    if (!this.cached) {
      // Render pending: show a loading placeholder
      wrapper.innerHTML = '<div class="cm-mermaid-loading">Rendering diagram...</div>';
    } else if ("error" in this.cached) {
      // Render error: show error message with source
      wrapper.innerHTML = `<div class="cm-mermaid-error">Mermaid error: ${escapeHtml(this.cached.error)}</div>`;
    } else {
      // Success: inject SVG
      wrapper.innerHTML = this.cached.svg;
      // Ensure SVG scales responsively
      const svg = wrapper.querySelector("svg");
      if (svg) {
        svg.style.maxWidth = "100%";
        svg.style.height = "auto";
      }
    }

    return wrapper;
  }

  get estimatedHeight(): number {
    // Reasonable default; SVGs vary in height
    return 200;
  }

  ignoreEvent(): boolean {
    return false;
  }
}
```

Key points:
- `estimatedHeight` returns 200 as a reasonable default. CM adjusts once the real DOM is measured, but having a non-zero estimate prevents layout jumps.
- Error display shows the mermaid parse error inline, which helps the user fix syntax without needing to open dev tools.
- The loading state only appears briefly while the async render runs.

#### 3d: StateField

```typescript
const mermaidRenderEffect = StateEffect.define<null>();

const mermaidDecoField = StateField.define<{ cursorLine: number; decos: DecorationSet }>({
  create(state) {
    const cursorLine = state.doc.lineAt(state.selection.main.head).number;
    return { cursorLine, decos: buildMermaidDecos(state, cursorLine, null) };
  },
  update(value, tr) {
    const cursorLine = tr.state.doc.lineAt(tr.state.selection.main.head).number;
    const hasRenderEffect = tr.effects.some(e => e.is(mermaidRenderEffect));
    if (!tr.docChanged && cursorLine === value.cursorLine && !hasRenderEffect) return value;
    return { cursorLine, decos: buildMermaidDecos(tr.state, cursorLine, tr.view) };
  },
  provide: (f) => EditorView.decorations.from(f, v => v.decos),
});
```

#### 3e: buildMermaidDecos

```typescript
function buildMermaidDecos(state: EditorState, cursorLine: number, view: EditorView | null): DecorationSet {
  const doc = state.doc;
  const fencedBlocks = scanFencedBlocks(doc);
  const mermaidBlocks = fencedBlocks.filter(b => b.lang === "mermaid");

  if (mermaidBlocks.length === 0) return Decoration.none;

  const builder = new RangeSetBuilder<Decoration>();

  for (const block of mermaidBlocks) {
    const cursorInBlock = cursorLine >= block.startLine && cursorLine <= block.endLine;

    if (cursorInBlock) {
      // Show raw source -- fenced code line decorations are already applied
      // by cmMarkdownDecorations.ts, so nothing extra needed here
      continue;
    }

    // Extract mermaid source (lines between opening and closing fences)
    const sourceLines: string[] = [];
    for (let ln = block.startLine + 1; ln < block.endLine; ln++) {
      sourceLines.push(doc.line(ln).text);
    }
    const source = sourceLines.join("\n").trim();
    if (!source) continue;

    // Trigger async render if not cached
    const cached = svgCache.get(source);
    if (!cached && view) {
      renderMermaid(source, view);
    }

    const from = doc.line(block.startLine).from;
    const to = doc.line(block.endLine).to;
    builder.add(from, to, Decoration.replace({
      widget: new MermaidWidget(source, cached),
      block: true,
    }));
  }

  return builder.finish();
}
```

#### 3f: Cache invalidation

When the document changes and a mermaid block's content changes, the old cache entry won't match (since the key is the source text), so a new render will trigger automatically. Old entries can accumulate. Add a simple eviction: when `svgCache.size > 50`, delete the oldest entries. This is a simple heuristic that prevents memory leaks without complex LRU tracking.

#### 3g: Theme awareness

The mermaid `theme` option should match the editor theme. Pass a `theme` parameter into the extension factory:

```typescript
export function mermaidDecorations(isDark: boolean): Extension {
  // Re-initialize mermaid with correct theme when theme changes
  mermaid.initialize({
    startOnLoad: false,
    theme: isDark ? "dark" : "default",
    securityLevel: "strict",
  });
  mermaidInitialized = true;
  svgCache.clear(); // Theme change invalidates all cached SVGs
  return [mermaidDecoField];
}
```

Since the editor is recreated when the theme changes (see the `useEffect` dependency array in `MarkdownEditor.tsx` line 219), this initialization will run fresh each time.

function MermaidPreviewBlock({ source }: { source: string }) {
  const ref = useRef<HTMLDivElement>(null);
  const [svgHtml, setSvgHtml] = useState<string>("");
  const [error, setError] = useState<string>("");

  useEffect(() => {
    let cancelled = false;
    ensureMermaidInit();
    const id = `mermaid-preview-${Date.now()}-${Math.random().toString(36).slice(2)}`;
    mermaid.render(id, source).then(
      ({ svg }) => { if (!cancelled) setSvgHtml(svg); },
      (e) => { if (!cancelled) setError(e instanceof Error ? e.message : String(e)); },
    );
    return () => { cancelled = true; };
  }, [source]);

  if (error) {
    return <div className="mermaid-error">Mermaid error: {error}</div>;
  }
  if (!svgHtml) {
    return <div className="mermaid-loading">Rendering diagram...</div>;
  }
  return <div className="mermaid-preview" ref={ref} dangerouslySetInnerHTML={{ __html: svgHtml }} />;
}

### Potential Challenges and Mitigations

1. **Mermaid bundle size**: The mermaid package is large (~2MB minified). Consider dynamic import (`await import("mermaid")`) so it's only loaded when a mermaid block is first encountered. This can be done in `ensureMermaidInit()` to keep the rest of the code synchronous.

2. **Async rendering in a synchronous StateField**: The StateField's `update` method is synchronous, but mermaid rendering is async. The pattern of using a `StateEffect` to trigger a rebuild after the async render completes handles this cleanly. The first pass shows a "Rendering..." placeholder; the second pass (after the effect) shows the SVG.

3. **Mermaid DOM requirements**: `mermaid.render()` may create temporary DOM elements. In a test environment (jsdom), this could fail. Tests should mock `mermaid.render` or focus on the decoration-building logic with pre-populated cache entries.

4. **Multiple rapid edits**: If the user types quickly inside a mermaid block, many async renders could fire. The `pendingRenders` set prevents duplicate renders for the same source. Additionally, when the cursor is inside the block, no rendering is triggered at all (the block shows raw text). Rendering only fires when the cursor leaves the block, at which point the source is stable.

5. **Height jumps**: When the SVG replaces the placeholder, the height changes. CM6 handles this through its height map update mechanism, but there may be a visible jump. The `estimatedHeight` of 200px provides a reasonable initial estimate to minimize this.
